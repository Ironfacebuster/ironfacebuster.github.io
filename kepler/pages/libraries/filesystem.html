filesystem

Dependencies: objects The filesystem library provides a set of functions for working with your filesystem.
This library takes advantage of Kepler's objects library, which provides a set of functions for working with objects.

fs_read

Reads a file, and stores its data as an object.

Required Parameters

Name Type Description
file String The path to the file.

Returns

String The file's internal object ID.

fs_exists

Checks if a file exists.

Required Parameters

Name Type Description
file String The path to the file.

Returns

Boolean True if the file exists, false otherwise.

fs_write

Writes data to a file, and creates it if it does not exist

Required Parameters

Name Type Description
file String The path to the file.
content Any The content to write to the file.

Returns

Int Error code, zero for success.

fs_append

Writes data to the end of a file, and creates it if it does not exist

Required Parameters

Name Type Description
file String The path to the file.
content Any The content to append to the file.

Returns

Int Error code, zero for success.
