Linking

In order to import libraries or other Kepler files, you have to link them.
Linking a file tells Kepler to load the file and import its contents, like variables and functions.

Importing a library

Kepler provides a few internal libraries to use, which can be imported into your code
To import an internal library, you need to link the library via its module name.

For example, this snippet imports the math library: linking.kep 1 start Header 2
link "math" 3 end Header 4 5 sin_of_twenty_three is call sin with 23.0 as value 6 print sin_of_twenty_three output -0.846220404175171

Linking Custom Files

To link a file you made, all you need to do is link a path to the file in the Header of your script.

For example, this snippet imports a file named "mylibrary.kep": linking.kep 1 start Header 2
link "./mylibrary.kep" 3 end Header As a result, the code within "mylibrary.kep" is run, and the variables and functions defined in it are available to the rest of your code. output This was printed from mylibrary.kep!
